## 2. 전체 시스템 한눈에 보기

```text
마이크 / WAV / 네트워크 오디오 입력
        ↓
16kHz mono PCM / float32 waveform 변환
        ↓
0.96초 window + 0.48초 hop 단위 분석
        ↓
YAMNet 추론
        ↓
521개 AudioSet class score 출력
        ↓
위험 소리 whitelist 추출
        ↓
threshold 적용
        ↓
debounce K/N 다수결
        ↓
cooldown 중복 알림 억제
        ↓
JSONL 로그 저장 + ESP32/임베디드 장치로 이벤트 전달
```

## 3. 먼저 알아야 할 핵심 결론

### 3.1 입력 형식이 틀리면 모델 결과도 틀어진다

YAMNet은 16kHz mono waveform을 전제로 한다. 입력이 44.1kHz이거나 stereo이거나 int16 정규화가 누락되면, 모델은 실행되더라도 의미 있는 결과를 내기 어렵다.

### 3.2 YAMNet score는 최종 판정이 아니다

YAMNet은 521개 클래스에 대한 score를 준다. 이 score를 그대로 알림으로 보내면 오탐과 중복 알림이 많아진다. 따라서 whitelist, threshold, debounce, cooldown이 필요하다.

### 3.3 위험 소리 감지는 일반 분류보다 recall이 중요하다

위험 소리를 놓치는 false negative는 실제 안전 문제로 이어질 수 있다. 따라서 threshold를 너무 높게 잡으면 위험하다. 다만 오탐이 너무 많으면 시스템 신뢰도가 떨어지므로 후처리로 균형을 잡아야 한다.

### 3.4 ESP32는 추론 장치가 아니라 센서/출력 장치로 보는 것이 현실적이다

ESP32는 I2S 마이크 입력, Wi-Fi 전송, 진동 모터 제어에 적합하다. YAMNet 추론은 Jetson, Raspberry Pi, PC 같은 더 강한 장치에서 수행하는 구조가 적절하다.

## 5. 프로젝트 구현 최소 성공 기준

아래가 모두 되면 최소 기능이 완성된 것으로 볼 수 있다.

- 16kHz mono 오디오를 입력받을 수 있다.
- 0.96초 window를 만들어 YAMNet에 넣을 수 있다.
- YAMNet scores에서 위험 클래스 score만 추출할 수 있다.
- threshold를 CLI 또는 설정 파일로 조정할 수 있다.
- debounce와 cooldown이 동작한다.
- 위험 이벤트가 JSONL로 저장된다.
- 위험 이벤트가 ESP32 또는 다른 출력 장치로 전달된다.
- 후처리 로직을 pytest로 단위 테스트할 수 있다.

## 6. 가장 먼저 해야 할 실습

```bash
python -m src.cli --input data/sample/test.wav --threshold 0.0 --verbose
```

이 명령으로 모든 후보 score를 먼저 확인한다. 그 다음 threshold를 0.4, 0.5, 0.6처럼 바꿔가며 어떤 클래스가 감지되는지 비교한다.
