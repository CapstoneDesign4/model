# 구현 / 디버깅 체크리스트

## 1. 입력 오디오 확인

- [ ] sample rate가 16kHz인가?
- [ ] mono인가?
- [ ] int16이면 float32로 정규화했는가?
- [ ] waveform shape이 1D인가?
- [ ] clipping이 발생하지 않는가?

```python
print(waveform.dtype)
print(waveform.shape)
print(waveform.min(), waveform.max())
```

## 2. YAMNet 추론 확인

- [ ] 모델이 정상 로드되는가?
- [ ] 최초 실행 시 TF-Hub 다운로드가 되는가?
- [ ] scores shape이 `(num_patches, 521)`인가?
- [ ] embeddings shape이 `(num_patches, 1024)`인가?
- [ ] 평균 score와 max score를 비교했는가?

## 3. Whitelist 확인

- [ ] `yamnet_class_map.csv`로 index를 직접 확인했는가?
- [ ] gunshot, siren, fire_alarm 등의 index가 정확한가?
- [ ] glass와 shatter 통합 정책을 정했는가?
- [ ] 클래스명을 로그에 사람이 읽기 쉽게 남기는가?

## 4. Threshold 확인

- [ ] threshold를 CLI에서 바꿀 수 있는가?
- [ ] 기본값 0.4를 실제 데이터로 검증했는가?
- [ ] 클래스별 threshold가 필요한지 확인했는가?
- [ ] threshold를 낮췄을 때 오탐이 얼마나 늘어나는가?
- [ ] threshold를 높였을 때 미탐이 얼마나 늘어나는가?

## 5. Debounce 확인

- [ ] K/N 다수결이 구현되어 있는가?
- [ ] 클래스별로 deque를 따로 관리하는가?
- [ ] transient sound에 너무 강한 debounce를 적용하지 않았는가?
- [ ] mock score로 단위 테스트했는가?

## 6. Cooldown 확인

- [ ] 동일 클래스 중복 이벤트가 억제되는가?
- [ ] 다른 클래스 이벤트까지 막지 않는가?
- [ ] cooldown 시간이 설정 파일에 있는가?
- [ ] 사이렌이 30초 지속될 때 알림 빈도가 적절한가?

## 7. 노이즈 캔슬링 확인

- [ ] NS 적용 전후 score를 비교했는가?
- [ ] 총소리/유리 파손 score가 떨어지지 않는가?
- [ ] aggressiveness가 너무 높지 않은가?
- [ ] NS를 끄고 켤 수 있는가?

## 8. 통신 확인

- [ ] 오디오 스트리밍은 UDP로 설계했는가?
- [ ] 위험 이벤트 전달은 TCP/UART처럼 안정적인 경로인가?
- [ ] sequence number가 있는가?
- [ ] heartbeat가 있는가?
- [ ] Wi-Fi 끊김 후 재연결되는가?

## 9. 로그 확인

- [ ] JSONL로 이벤트를 저장하는가?
- [ ] timestamp, class, score, seq가 들어가는가?
- [ ] 오류 발생 시 원인을 추적할 수 있는 로그가 있는가?
- [ ] 로그 파일 크기 증가에 대한 정책이 있는가?

## 10. 운영 확인

- [ ] systemd로 실행할 수 있는가?
- [ ] Restart=always가 설정되어 있는가?
- [ ] journalctl로 로그 확인이 가능한가?
- [ ] 장시간 실행 시 메모리 사용량이 안정적인가?
- [ ] CPU 사용률과 발열을 확인했는가?

## 11. 문제별 빠른 진단

### 아무 감지도 안 될 때

1. 마이크 입력 확인
2. waveform min/max 확인
3. sample rate 확인
4. threshold를 0.0으로 낮춰 실행
5. whitelist index 확인
6. debounce 조건 완화

### 오탐이 많을 때

1. threshold 조정
2. debounce 강화
3. cooldown 확인
4. 오탐 상황의 로그 분석
5. 노이즈 캔슬링 약하게 적용 실험

### 알림이 계속 반복될 때

1. cooldown 적용 여부 확인
2. class별 cooldown인지 확인
3. ESP32 출력 로직 중복 실행 확인
4. 동일 이벤트 재전송 여부 확인

### 네트워크 통신이 안 될 때

1. IP/port 확인
2. 같은 네트워크인지 확인
3. 방화벽 확인
4. UDP/TCP 구분 확인
5. heartbeat 수신 여부 확인
